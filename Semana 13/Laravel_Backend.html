1. Servidor Web: Apache HTTP Server 🌐

El Apache HTTP Server, o simplemente "Apache", es el servidor web de código abierto más popular y utilizado en el mundo. Su función principal es entregar contenido web a los usuarios.

Cuando un usuario escribe una URL en su navegador, envía una "solicitud" a un servidor. Apache es el software que reside en ese servidor y se encarga de:

  • Recibir request HTTP/HTTPS del navegador.
  • Buscar el contenido solicitado HTML (y imágenes, videos, archivos PDF, etc.) en el servidor.
  • Enviar response (ese contenido) de vuelta al navegador del usuario.

a. Apache Lounge

Apache Lounge es un sitio web y una comunidad que se especializa en proporcionar versiones compiladas (binarios) del Apache HTTP Server, principalmente para sistemas operativos Windows. Permite ejecutar Apache Lounge en Windows directamente desde la versión ZIP sin instalarlo como servicio.

b. Instalación del Servidor Portable

  1. Descarga: Visita apachelounge.com/download/ y descarga el ZIP adecuado (ej., `httpd-2.4.xx-win64-VS17.zip`).
  2. Extrae el ZIP: Descomprime el archivo descargado (por ejemplo, `httpd-2.4.xx-win64-VS17.zip`) en una carpeta como `D:\Server\Apache24`.
  3. Verifica dependencias: Asegúrate de tener instalado el paquete de Visual C++ Redistributable correspondiente (por ejemplo, VC15 o VC17). Si no, puedes descargarlo desde el sitio de Apache Lounge.
  4. Configura `httpd.conf`: Abre `D:\server\Apache24\conf\httpd.conf` y revisa que tengas las siguientes directivas: 
    Listen 8080
ServerName localhost:8080
Define SRVROOT "D:\server\Apache24"
  5. Ejecutar el Servidor: Desde la línea de comandos, navega a `D:\server\Apache24\bin` y ejecuta: 
    httpd.exe
  6. Visualizar: Abre tu navegador y visita `http://localhost:8080`. Deberías ver "It works!".

c. Archivo de Configuración "httpd.conf"

El archivo "httpd.conf" es el corazón de la configuración de Apache HTTP Server. Contiene directivas que dictan cómo se comporta el servidor, desde qué puertos escucha hasta cómo maneja las solicitudes y qué contenido sirve. Algunas directivas principales son:

  • `ServerRoot "ruta"`: Define el directorio base donde está instalada la configuración y los módulos de Apache.
  • `Listen [IP:]puerto`: Indica a Apache en qué direcciones IP y puertos debe escuchar las solicitudes entrantes.
  • `ServerName FQDN[:puerto]`: Establece el nombre de host y puerto que el servidor usa para identificarse a sí mismo.
  • `DocumentRoot "ruta"`: Especifica el directorio principal desde donde Apache sirve los archivos web por defecto.
  • `DirectoryIndex nombre_archivo`: Define el archivo que Apache buscará y servirá por defecto cuando se solicita un directorio.
  • `ErrorLog "ruta"`: Define la ubicación del archivo donde Apache registra los errores del servidor.
  • `LogLevel nivel`: Controla el nivel de detalle de los mensajes registrados en el ErrorLog.
  • `LoadModule nombre_modulo`: Carga módulos adicionales para extender la funcionalidad de Apache.

2. PHP - Hypertext Preprocessor 🐘

PHP (acrónimo recursivo de PHP: Hypertext Preprocessor) es un lenguaje de programación de código abierto del lado del servidor, diseñado específicamente para el desarrollo web. Es uno de los lenguajes más utilizados en el mundo para construir sitios y aplicaciones web dinámicas.

a. ¿Cómo Funciona PHP?

Cuando solicitas una página con código PHP, el servidor pasa ese código a un intérprete PHP. Este intérprete procesa las instrucciones, que pueden incluir interactuar con bases de datos, manejar datos de formularios o generar contenido dinámico. El resultado de este procesamiento es código HTML, CSS y JavaScript puro, que el servidor luego envía a tu navegador. Tu navegador solo recibe y muestra este HTML final, sin ver nunca el código PHP original, haciendo que las páginas web sean dinámicas e interactivas.

b. Instalación de PHP

  1. Descarga el ZIP oficial: Ve a windows.php.net/download/ y elige la versión para tu arquitectura (x64 o x86). Asegúrate de descargar el archivo ZIP, no el instalador. Para Apache, se recomienda usar las versiones Thread Safe.
  2. Extrae el contenido: Descomprime el archivo ZIP en una carpeta como `D:\server\php`.
  3. Configura `php.ini`: Dentro de `D:\server\php`, copia `php.ini-development` y renómbralo como `php.ini`. Abre el archivo `php.ini` con un editor de texto y habilita extensiones necesarias quitando el `;` al inicio de las líneas, por ejemplo: 
    extension_dir = "ext"
extension=mysqli
extension=curl
  4. Agrega PHP al PATH: Ve a Configuración del sistema > Variables de entorno. En "Path", agrega la ruta `D:\server\php`. Reinicia la consola para que los cambios surtan efecto.
  5. Verifica la instalación: Abre una nueva consola (CMD) y escribe: 
    php -v

c. Sintaxis Básica de PHP

  • Estructura básica: Todo código PHP debe ir entre etiquetas especiales ``. 
    // Tu código PHP aquí
echo "Hola, mundo!"; // Ejemplo
  • Comentarios: Puedes comentar tu código usando `//` o `#` para una línea, y `/* ... */` para varias líneas.
  • Variables: Se declaran con el signo `$` (ej., `$nombre = "Ana";`). Las variables sí distinguen entre mayúsculas y minúsculas. 
    ?php
$nombre = "Ana";
$edad = 25;
?>
  • Salida de datos: `echo` y `print` se utilizan para mostrar información en pantalla. `echo` es un poco más rápido. 
    ?php
echo "¡Hola, mundo!";
print "PHP es genial";
?>
  • Control de flujo condicional:
    • `if`, `else` y `elseif`: Permiten ejecutar bloques de código basados en condiciones. 
      = 18) {
    echo "Eres mayor de edad.";
} elseif ($edad == 17) {
    echo "Casi llegas.";
} else {
    echo "Eres menor de edad.";
}
?>
    • `switch`: Útil cuando se evalúa una misma variable contra diferentes valores. 
      ?php
$color = "azul";
switch ($color) {
    case "rojo":
        echo "El color es rojo.";
        break;
    case "azul":
        echo "El color es azul.";
        break;
    default:
        echo "Color no reconocido.";
}
?>
  • Estructuras de repetición (bucles):
    • `for`: Ejecuta un bloque de código un número definido de veces. 
      ?php
for ($i=0; $i<5; $i++) {
    echo "Número: " . $i . "
      ";
}
?>
    • `while` y `do...while`: Se utilizan para repetir un bloque de código mientras se cumpla una condición. El `do...while` se ejecuta al menos una vez. 
      ?php
$x = 0;
while ($x < 5) {
    echo "Valor: " . $x . "
      ";
    $x++;
}

$y = 0;
do {
    echo "Valor: " . $y . "
      ";
    $y++;
} while ($y < 5);
?>
    • `foreach`: Especialmente útil para recorrer arrays, ya que itera sobre cada elemento del array. 
      ?php
$frutas = ["Manzana", "Banana", "Cereza"];
foreach ($frutas as $fruta) {
    echo "Fruta: " . $fruta . "
      ";
}
?>
  • Manejo de archivos e inclusión: `include`, `include_once`, `require` y `require_once` permiten incorporar archivos externos. `include` genera un `warning` si el archivo no se encuentra pero continúa la ejecución, mientras que `require` detiene la ejecución con un `fatal error`. 
    ?php
include 'funciones.php';
require 'config.php';
?>
  • Funciones: Puedes encapsular bloques de código en funciones para reutilizarlas y organizar tu programa. 
    ?php
function saludar ($nombre) {
    return "¡Hola, " . $nombre . "!";
}
echo saludar("Carlos");
?>
  • Finalización del script: `exit` o `die` detienen la ejecución del script. 
    ?php
if (!file_exists("importante.txt")) {
    exit("El archivo importante no se encontró.");
}
?>

Estas instrucciones y estructuras son la base para desarrollar en PHP, permitiéndote construir desde scripts simples hasta aplicaciones web complejas.

3. Gestión de Dependencias con Composer 📦

Composer es la herramienta fundamental y el gestor de dependencias de facto para PHP. Funciona de manera similar a otras herramientas como npm (Node.js) o pip (Python). Composer no instala paquetes globalmente en tu sistema; en su lugar, instala las dependencias por cada proyecto en un directorio local (generalmente `vendor/`).

a. Descargar e Instalar Composer

  1. Descarga: Visita getcomposer.org/download/.
  2. Requisitos previos: Asegúrate de tener PHP instalado y agregado al PATH del sistema. Puedes verificarlo abriendo la consola (CMD) y escribiendo: 
    php -v
  3. Ejecuta el instalador: Abre el archivo `Composer-Setup.exe` descargado. Elige la ruta del ejecutable de PHP (por ejemplo, `D:\php\php.exe`), acepta las opciones por defecto y finaliza la instalación.
  4. Verifica la instalación: Abre una nueva consola y escribe: 
    composer -V

4. Laravel Framework ✨

Laravel es un framework de PHP de código abierto que se centra en la simplicidad, la elegancia y el uso de patrones modernos de desarrollo. Se apoya en el patrón Modelo-Vista-Controlador (MVC) para organizar la lógica de la aplicación, lo que facilita el mantenimiento del código y la separación de responsabilidades. Laravel framework es muy extenso y su comunidad crece día a día.

  • Link: laravel.com
  • Documentación: laravel.com/docs/12.x

a. Terminología Laravel

  • Rutas (Routes): Son los puntos de entrada de la aplicación. Asignan una URL a una lógica específica (por ejemplo, a un controlador o a un `closure`) para responder a las solicitudes HTTP.
  • Controladores (Controllers): Agrupan la lógica de negocio y gestionan las peticiones del usuario. Reciben la solicitud, interactúan con los modelos y retornan una respuesta.
  • Vistas y Blade: Blade es el motor de plantillas de Laravel para separar la presentación del contenido y la lógica. Facilita la incorporación de código PHP en HTML mediante directivas específicas.
  • Eloquent ORM: Es la herramienta que ofrece Laravel para interactuar con la base de datos. Mapea las tablas a clases `modelo` permitiendo trabajar con datos mediante una sintaxis intuitiva para operaciones CRUD.
  • Middleware: Son filtros o capas intermedias que se sitúan entre la solicitud y la respuesta. Se utilizan para tareas como la autenticación, autorización o verificación de condiciones antes de acceder a ciertas rutas.
  • Artisan: Es la interfaz de línea de comandos de Laravel. Con Artisan se automatizan tareas comunes como la generación de código, ejecución de pruebas y administración de la base de datos.
  • Migrations y Seeds: Las migraciones permiten versionar y modificar la estructura de la base de datos de forma programática. Los seeds son scripts destinados a poblar la base de datos con datos iniciales o de prueba.
  • Service Providers: Constituyen el mecanismo central de Laravel para registrar y configurar los servicios y componentes que forman la aplicación. A través de ellos se cargan aspectos fundamentales como la base de datos, el sistema de colas, etc.
  • Facades: Ofrecen una interfaz estática para clases y servicios registrados en el contenedor de servicios de Laravel, simplificando la sintaxis.
  • Composer: Es el gestor de dependencias con el que se instala y actualiza el framework, junto con sus paquetes y bibliotecas.

b. Instalación de Laravel con Composer

Requisitos previos: PHP (versión 8.1 o superior), Composer y Node.js y NPM (opcional, pero útil para compilar assets). Verifica que las extensiones `zip`, `fileinfo`, `pdo_sqlite` y `pgsql` estén habilitadas en tu `php.ini`.

  1. Instalar con Composer: Ejecuta el siguiente comando dentro de `htdocs/laravel` (si usas XAMPP/WAMP): 
    composer create-project laravel/laravel nombre-de-tu-proyecto
  2. Accede al directorio: 
    cd nombre-de-tu-proyecto
  3. Inicia el servidor de desarrollo: 
    php artisan serve
  4. Luego abre tu navegador en `http://localhost:8000`.

5. Estructura de Directorios de Laravel 📁

La estructura de directorios en un proyecto Laravel está pensada para organizar el código de manera lógica, facilitando tanto el desarrollo como el mantenimiento de la aplicación.

  • Raíz del proyecto: Aquí se encuentran archivos fundamentales como `composer.json`, `artisan` (la herramienta de línea de comandos de Laravel) y otros archivos de configuración. Este directorio es el punto de partida del proyecto.
  • `app`: Es el núcleo de la aplicación y contiene la lógica de negocio. Dentro de `app` se suelen encontrar subdirectorios como:
    • `Console`: Contiene los comandos personalizados de Artisan.
    • `Http`: Aquí se alojan los controladores, middleware, y formularios (requests).
    • `Providers`: Registra los proveedores de servicios de la aplicación.
    • `Models` (opcional): Algunos proyectos prefieren una carpeta específica para ellos.
  • `bootstrap`: Contiene el archivo `app.php`, encargado de iniciar el framework, y un directorio `cache` para almacenar archivos generados que ayudan a la optimización del rendimiento.
  • `config`: Aquí se encuentran todos los archivos de configuración de la aplicación, abarcando desde la base de datos hasta la configuración de servicios externos y la gestión de la sesión.
  • `database`: Aloja las migraciones (para versionar la base de datos), los factories (para generar datos de prueba) y los seeders (para poblar la base de datos con datos iniciales). También puede contener una base de datos SQLite.
  • `public`: Es el punto de entrada para todas las solicitudes HTTP, ya que contiene el archivo `index.php`. Además, se alojan los activos públicos como imágenes, hojas de estilo (CSS) y archivos JavaScript.
  • `resources`: Aquí se gestionan los elementos que serán compilados o transformados, como las vistas (Blade), archivos de lenguaje y recursos sin compilar como CSS o JavaScript.
  • `routes`: En esta carpeta se definen todas las rutas de la aplicación. Por defecto, encontrarás archivos como:
    • `web.php`: Incluye las rutas que conforman la interfaz de la aplicación web.
    • `api.php`: Contiene rutas orientadas a una API RESTful.
    • `console.php`: Para comandos basados en la línea de comandos.
  • `storage`: Este directorio se encarga de almacenar archivos generados por la aplicación, como logs, cachés, sesiones y archivos subidos. Se organiza en varias subcarpetas (`app`, `framework`, `logs`).
  • `tests`: Contiene los tests unitarios y de funcionalidad que facilitan el aseguramiento de la calidad del código y la confiabilidad de la aplicación.
  • `vendor`: Es administrado por Composer y agrupa todas las dependencias del proyecto, incluyendo los paquetes de terceros necesarios.
