Desarrollo Web Backend

Fundamentos con Java, Spring y Jakarta: Arquitectura y Herramientas Esenciales

Conceptos Clave

Apache Tomcat

Contenedor de servlets desarrollado bajo el proyecto Jakarta, implementa especificaciones de servlets y JSP, funcionando como un servidor web para aplicaciones Java.

Estructura de Directorios:

  • `bin`: Ejecutables y scripts.
  • `conf`: Archivos de configuración.
  • `webapps`: Aplicaciones web.
  • Y más...

Configuración server.xml

Núcleo de la configuración de Apache Tomcat, define la estructura y parámetros clave del servidor y sus componentes.

Elementos Clave:

  • `Server`, `Service`, `Connector`
  • `Engine`, `Host`, `Context`
  • Parámetros: `port`, `maxThreads`, etc.

Gestión de Dependencias (Maven)

Maven automatiza la administración de librerías y módulos, estandarizando estructuras de proyectos Java y facilitando la construcción del software.

Clave:

  • `pom.xml`: Archivo de configuración del proyecto.
  • Repositorio Maven: Centraliza las librerías.
  • Ciclo de vida de construcción (compile, package, install).

Jakarta Server Pages (JSP)

Lenguaje de script del lado del servidor para crear contenido web dinámico e independiente de la plataforma, con acceso a las API de Java.

Procesamiento:

  • JSP se transforma en un servlet.
  • El servlet se compila y ejecuta.
  • Genera HTML para el cliente.

Sintaxis JSP

Elementos clave para insertar lógica Java y mostrar datos en las páginas JSP.

Elementos:

  • Scriptlets (`<% ... %>`): Bloques de código Java.
  • Declaraciones (`<%! ... %>`): Variables/métodos de la clase servlet.
  • Expresiones (`<%= ... %>`): Imprimen valores directamente.

Directivas y Acciones JSP

Elementos para controlar el comportamiento y la inclusión de recursos en las páginas JSP.

Tipos:

  • Directivas (`<%@ ... %>`): `page`, `include`, `taglib`.
  • Acciones (``): `jsp:include`, `jsp:useBean`, etc.

Objetos Implícitos JSP

Nueve variables predefinidas automáticamente disponibles en todas las páginas JSP para acceder a recursos y funcionalidades del servidor.

Ejemplos:

  • `request`, `response`, `out`
  • `session`, `application`, `pageContext`
  • `config`, `page`, `exception`

MVC en JSP

Patrón arquitectónico que separa la aplicación en tres componentes interconectados para mejorar la modularidad y el mantenimiento.

Componentes:

  • Modelo: Lógica de negocio y datos.
  • Vista: Presentación (JSP).
  • Controlador: Maneja las interacciones del usuario.

Spring Framework

Robusto framework Java para el desarrollo de aplicaciones empresariales, diseñado para estandarizar y agilizar la programación, con una gran comunidad.

Conceptos Clave:

  • Bean: Componente manejado por Spring.
  • Inversión de Control (IoC): El framework controla el ciclo de vida de los objetos.

Inyección de Dependencia (DI)

Tipo de IoC donde las propiedades de un objeto son inyectadas a través de constructor, setter o campo, mejorando la modularidad.

Tipos:

  • Basada en Constructor.
  • Basada en Setter.
  • Basada en Campo (evitar fuera del contenedor).

Spring Container

Contenedor de Inversión de Control (IoC Container) que gestiona el ciclo de vida completo de los objetos (beans) de la aplicación.

Función:

  • Crea, relaciona, configura y destruye beans.
  • Realiza la inyección de dependencia.
  • Utiliza metadatos (ej., XML) para la configuración.

Spring Boot

Herramienta que simplifica la creación de proyectos con Spring Framework, eliminando la complejidad de las configuraciones y dependencias.

Características:

  • Configuración automática.
  • Servidor incorporado.
  • Métricas y comprobación de "salud".
