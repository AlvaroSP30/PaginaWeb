React: La Librería Declarativa para Construir Interfaces de Usuario

Componentes, estado, props y Virtual DOM.

Contenidos del Artículo

Introducción a React

React (a menudo llamado React.js o ReactJS) es una librería de JavaScript de código abierto y declarativa para construir interfaces de usuario (UI) interactivas y eficientes. Fue creada por Facebook (ahora Meta) y es mantenida por una comunidad de desarrolladores. React se centra en la idea de componentes reutilizables, lo que facilita el desarrollo, la gestión y la escalabilidad de aplicaciones web complejas, especialmente las Single Page Applications (SPAs).

1. Origen e Historia Breve

  • Creador: Jordan Walke (ingeniero de software en Facebook).
  • Año de lanzamiento inicial: 2013.
  • Propósito inicial: Resolver problemas de rendimiento y complejidad en las interfaces de usuario de Facebook (específicamente, el News Feed).
  • Influencias: Fue influenciado por lenguajes funcionales y la idea de componentes.
  • Ecosistema: Aunque es una librería para UI, su popularidad ha llevado a la creación de un vasto ecosistema de herramientas, otras librerías y frameworks que lo complementan (como React Router para enrutamiento, Redux para gestión de estado, Next.js para renderizado del lado del servidor, etc.).

2. ¿Por qué usar React?

  • Declarativo: Describes cómo quieres que se vea tu UI y React se encarga de las actualizaciones del DOM.
  • Basado en Componentes: La aplicación se construye a partir de pequeños, aislados y reutilizables bloques de código.
  • Aprende una vez, escribe en cualquier lugar: React Native permite escribir aplicaciones móviles nativas.
  • Virtual DOM: React utiliza una representación ligera en memoria del DOM real. Solo aplica los cambios mínimos y necesarios al DOM, optimizando el rendimiento.
  • Gran Comunidad: Acceso a abundante documentación, tutoriales y soporte.

3. Conceptos Fundamentales

a. Componentes

Son los bloques de construcción de cualquier aplicación React. Encapsulan lógica, estado y la interfaz de usuario.

  • Componentes de Clase (menos comunes hoy en día): 
    import React from 'react';

class Saludo extends React.Component {
    render() {
        return <h1>Hola, {this.props.nombre}!</h1>;
    }
}
// Uso: <Saludo nombre="Mundo" />
  • Componentes Funcionales (preferidos con Hooks): 
    import React from 'react';

function SaludoFuncional(props) {
    return <h1>Hola, {props.nombre}!</h1>;
}
// Uso: <SaludoFuncional nombre="Amigo" />

b. JSX (JavaScript XML)

Una extensión de la sintaxis de JavaScript para describir la UI. Permite escribir HTML-like en JS.

  • Con JSX (lo que escribes): 
    <h1>Hola Mundo</h1>
  • Incorporando lógica JS con llaves {}: 
    const nombre = "React";
const elemento = <h1>Hola, {nombre}!</h1>;
  • Reglas clave: Un solo elemento raíz, atributos en camelCase (ej. className), eventos en camelCase (ej. onClick).

c. Props (Propiedades)

Argumentos que se pasan a los componentes desde su componente padre. Son de solo lectura (inmutables).

function Boton({ texto, onClick }) { // Destructuring de props
    return <button onClick={onClick}>{texto}</button>;
}

// Uso
<Boton texto="Click Aquí" onClick={() => alert('¡Botón clickeado!')} />

d. Estado (State)

Datos internos de un componente que pueden cambiar con el tiempo. Cuando el estado cambia, React re-renderiza el componente.

import React, { useState } from 'react';

function Contador() {
    const [cuenta, setCuenta] = useState(0); // [estado actual, función para actualizar estado]

    const incrementar = () => {
        setCuenta(cuenta + 1);
    };

    return (
        <div>
            <p>La cuenta es: {cuenta}</p>
            <button onClick={incrementar}>Incrementar</button>
        </div>
    );
}

Importante: Siempre usa la función de actualización (ej. setCuenta) para modificar el estado.

e. Ciclo de Vida de los Componentes

Las diferentes fases por las que pasa un componente. En componentes funcionales, los **Hooks** (especialmente useEffect) manejan estas fases.

f. Hooks (para componentes funcionales)

Funciones que permiten "engancharse" al estado y características del ciclo de vida desde componentes funcionales.

  • useState: Para añadir estado.
  • useEffect: Para efectos secundarios (llamadas a API, suscripciones, etc.). 
    import React, { useState, useEffect } from 'react';

function FetchData() {
    const [data, setData] = useState(null);
    const [loading, setLoading] = useState(true);

    useEffect(() => {
        fetch('https://jsonplaceholder.typicode.com/todos/1')
            .then(response => response.json())
            .then(json => {
                setData(json);
                setLoading(false);
            });
        return () => { console.log("Limpieza del efecto"); };
    }, []); // Array de dependencias vacío: se ejecuta solo una vez al montar

    if (loading) return <p>Cargando datos...</p>;
    return <p>Datos: {data.title}</p>;
}
  • Otros Hooks comunes: useContext, useReducer, useRef, useCallback, useMemo.

g. Renderizado Condicional

Permite renderizar diferentes elementos o componentes basados en una condición.

function MensajeLogin(props) {
    const estaLogueado = props.estaLogueado;
    if (estaLogueado) {
        return <p>Bienvenido de nuevo!</p>;
    }
    return <p>Por favor, inicia sesión.</p>;
}

h. Listas y Keys

Al renderizar una lista, React requiere una prop key única para cada elemento, lo que ayuda a optimizar el rendimiento.

function ListaElementos(props) {
    const items = props.items;
    return (
        <ul>
            {items.map(item =>
                <li key={item.id}>
                    {item.nombre}
                </li>
            )}
        </ul>
    );
}

4. Flujo de Datos Unidireccional (One-Way Data Flow)

Los datos fluyen en una sola dirección: de los componentes padres a los hijos a través de las props. Esto hace que el flujo sea predecible y la depuración más sencilla.

5. Conceptos Avanzados y Ecosistema

  • Context API: Para compartir datos globales en un árbol de componentes sin pasar props manualmente.
  • Redux / Zustand / Jotai: Librerías para la gestión de estado más complejas.
  • React Router: Librería estándar para gestionar el enrutamiento en SPAs.
  • Next.js / Gatsby: Frameworks que extienden React con SSR, SSG y optimizaciones.
  • Testing: Jest y React Testing Library son herramientas comunes.
  • Herramientas de Desarrollo: La extensión "React Developer Tools" para navegadores.

6. ¿Cómo empezar con React?

La forma más común es usar Vite o Create React App:

  • Con Vite: npm create vite@latest my-react-app -- --template react
  • Con Create React App: npx create-react-app my-react-app

Estos comandos configuran un proyecto de React con una estructura básica y herramientas de desarrollo como Babel y Webpack.
