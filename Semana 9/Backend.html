Fundamentos del desarrollo Backend

Los fundamentos del desarrollo Backend son base esencial para construrir aplicaciones web más robustas y eficientes.

1. Arquitectura de las aplicaciones Web

  • Arquitectura Cliente - Servidor: Se explica el modelo fundamental donde los clientes (navegadores) envían peticiones a un servidor web y un servidor de base de datos, quienes procesan y devuelven respuestas. Se enfatiza la división entre Frontend (Side-Client) y Backend (Side-Server).
  • Tipos de Arquitectura
    • Arquitectura C/S Clásica - Multiples Páginas: Se describe como el modelo tradicional donde cada página es un archivo HTML, CSS y JS independiente, con navegación a través de enlaces.
    Arquitectura C/S Multicapa: Se introduce como una técnica de lado del servidor para organizar sistemas complejos en capas, separando responsabilidades (ej. Modelo MVC).

2. Autenticación basada en tokens

  • Concepto: Un método de verificación de identidad que permite a los usuarios acceder a sistemas sin necesidad de proporcionar credenciales constantemente. Se basa en la emisión de un "token" (datos cifrados) que valida la identidad del usuario por un período determinado.
  • Fundamentos Claves:
    • Generación de tokens: Se crea un token único al iniciar sesión.
    • Uso de tokens: El usuario incluye el token en las peticiones al servidor en lugar de credenciales.
    • Validación de tokens: El servidor verifica la validez y vigencia del token.
    • Expiración y Renovación: Los tokens tienen duración limitada y requieren renovación.
    • Seguridad: Los tokens están cifrados y firmados digitalmente para prevenir manipulación.
  • Tipos de token:
    • JSON Web Token (JWT): mpliamente utilizado en aplicaciones web y móviles.
    • OAuth Tokens: Usados en autenticación delegada para acceder a APIs.
    • SAML Tokens: Comunes en entornos empresariales.

3. Descompresión de Archivos Zips

  • Componentes de Clase (menos comunes hoy en día): Se incluye una instrucción práctica sobre cómo descomprimir archivos zip, un paso común en la gestión de recursos o dependencias en proyectos de desarrollo.
