Principios Fundamentales de React y su Instalación

Entendiendo React y cómo iniciar tu primer proyecto.

Principios Fundamentales de React

React se basa en un conjunto de principios clave que guían su diseño y su forma de trabajar con las interfaces de usuario. Comprenderlos es crucial para escribir código React efectivo y escalable.

1. Declarativo vs. Imperativo

  • Declarativo: En React, tú describes qué quieres que se vea la UI en un momento dado, en función del estado de tus datos. React se encarga de averiguar cómo realizar los cambios en el DOM para que la UI coincida con tu descripción. 
    // Enfoque Declarativo (React):
// describes el resultado final
function MiComponente({ valor }) {
    return <p>El valor es: {valor}</p>;
}
// React actualiza automáticamente el DOM cuando 'valor' cambia.
  • Imperativo: Con manipulación directa del DOM (JavaScript puro), le das instrucciones paso a paso a la computadora sobre cómo cambiar la UI. 
    // Enfoque Imperativo (JS puro):
// instrucciones paso a paso para el DOM
const miParrafo = document.getElementById('miParrafo');
if (algunEstado) {
    miParrafo.textContent = 'Nuevo Valor';
    miParrafo.style.color = 'blue';
} else {
    miParrafo.textContent = 'Valor Inicial';
    miParrafo.style.color = 'black';
}
  • Ventaja React: Al ser declarativo, el código es más fácil de entender, depurar y predecir, ya que solo te preocupas por el estado deseado.

2. Basado en Componentes

  • Todo en React es un componente. Una aplicación se construye ensamblando componentes pequeños e independientes.
  • Cada componente tiene su propia lógica y apariencia, y puede ser reutilizado en diferentes partes de la aplicación.
  • Esto fomenta la modularidad, la reutilización del código y una mejor organización del proyecto.
  • Los componentes pueden ser funciones (preferido con Hooks) o clases.

3. Flujo de Datos Unidireccional (One-Way Data Flow)

  • Los datos en React fluyen en una sola dirección: de un componente padre a sus componentes hijos a través de las props (propiedades).
  • Los componentes hijos no pueden modificar directamente las props que reciben de sus padres; son de solo lectura.
  • Si un componente hijo necesita comunicar un evento o un cambio al padre, lo hace invocando una función (callback) que el padre le pasó como prop.
  • Este principio hace que el flujo de datos sea predecible y facilita mucho el seguimiento de errores ("debugging").

4. Virtual DOM

  • React no manipula el DOM real directamente cada vez que hay un cambio de estado. En su lugar, utiliza un Virtual DOM, que es una representación ligera en memoria del DOM real.
  • Cuando el estado de un componente cambia, React:
    1. Crea una nueva versión del Virtual DOM.
    2. Compara (o "difiere") esta nueva versión con la anterior para identificar solo las diferencias.
    3. Actualiza el DOM real de manera eficiente, aplicando solo los cambios mínimos y necesarios.
  • Este proceso, llamado **reconciliación**, es extremadamente rápido y optimiza el rendimiento, ya que la manipulación directa del DOM es una operación costosa para el navegador.

5. "Aprende una vez, escribe en cualquier lugar"

  • La filosofía de React es que los conocimientos y patrones aprendidos en React para la web son aplicables a otras plataformas.
  • Con React Native, puedes usar los mismos principios y una sintaxis similar para construir aplicaciones móviles nativas para iOS y Android.

Cómo Instalar y Crear un Proyecto React

Para empezar a desarrollar con React, necesitas tener Node.js y npm (o Yarn) instalados en tu sistema. Estas herramientas son esenciales para gestionar las dependencias del proyecto y ejecutar scripts de desarrollo.

Paso 1: Verificar Node.js y npm/Yarn

Abre tu terminal o línea de comandos y ejecuta los siguientes comandos para verificar si ya los tienes instalados:

node -v
npm -v
# O si prefieres Yarn:
yarn -v

Si no están instalados, visita la página oficial de Node.js para descargar e instalar el paquete que incluye npm. Para Yarn, visita Yarnpkg.com.

Paso 2: Elegir una Herramienta de Andamiaje (Scaffolding Tool)

No se instala React directamente con npm install react para iniciar un proyecto completo. En su lugar, se utilizan herramientas que configuran un entorno de desarrollo moderno con todas las dependencias necesarias (Babel para JSX, Webpack para bundling, etc.). Las opciones más populares son:

  • Vite (Recomendado actualmente): Un constructor de proyectos muy rápido y ligero, ideal para proyectos modernos.
  • Create React App (CRA): La herramienta oficial y tradicional de Facebook, más robusta pero a menudo más lenta para iniciar.

Paso 3: Crear tu Proyecto React

Opción A: Usando Vite (Recomendado)

Vite es más rápido y eficiente. Abre tu terminal en la carpeta donde deseas crear tu proyecto y ejecuta:

npm create vite@latest mi-primer-react-app -- --template react
  • npm create vite@latest: Llama al ejecutable de Vite para crear un proyecto.
  • mi-primer-react-app: Es el nombre de tu proyecto (puedes cambiarlo). Se creará una nueva carpeta con este nombre.
  • -- --template react: Indica a Vite que use la plantilla de React.

Luego, navega a la carpeta del proyecto e instala las dependencias:

cd mi-primer-react-app
npm install

Opción B: Usando Create React App (CRA)

Si prefieres la herramienta tradicional, usa el siguiente comando:

npx create-react-app mi-primer-react-app
  • npx: Ejecuta un paquete de npm sin instalarlo globalmente.
  • create-react-app: Es el comando para generar el proyecto.
  • mi-primer-react-app: El nombre de tu proyecto.

Este comando creará la carpeta y las dependencias automáticamente.

Paso 4: Iniciar el Servidor de Desarrollo

Una vez que las dependencias estén instaladas (o que CRA haya terminado), navega a la carpeta de tu proyecto (si usaste Vite) y ejecuta el comando para iniciar el servidor de desarrollo:

# Si usaste Vite:
cd mi-primer-react-app
npm run dev

# Si usaste Create React App:
cd mi-primer-react-app
npm start

Esto abrirá tu aplicación React en el navegador (generalmente en `http://localhost:5173/` para Vite o `http://localhost:3000/` para CRA) y podrás ver tu aplicación React funcionando. Cualquier cambio que hagas en el código se reflejará automáticamente gracias al "Hot Module Replacement".
