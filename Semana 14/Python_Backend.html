Desarrollo Web Backend con Python y Flask

Una introducción al poder del lado del servidor

1. Teoría y Conceptos Clave 🐍

El desarrollo web **backend** se encarga de la **lógica del lado del servidor**, la gestión de bases de datos y la comunicación entre el navegador (frontend) y la información. Cuando usas Python y Flask, estás trabajando en este "detrás de escenas" de una aplicación web.

a. ¿Qué es Flask?

Flask es un **microframework** de Python para el desarrollo web. Es "micro" porque no incluye herramientas o librerías por defecto para todo (como bases de datos o validación de formularios), lo que lo hace muy ligero y flexible. Tú decides qué componentes añadir, lo que lo convierte en una excelente opción para construir APIs RESTful, aplicaciones web pequeñas y medianas, o microservicios.

b. Conceptos Fundamentales:

  • Rutas (Routes): Son las URL que tu aplicación "escucha". Cuando un usuario visita una URL específica (ej., / para la página de inicio, o /usuarios), Flask sabe qué función de Python debe ejecutar. Se definen con el decorador @app.route(). 
    @app.route('/')
def index():
    return "¡Hola, mundo desde Flask!"
  • Vistas (View Functions): Son las funciones de Python que se ejecutan cuando se accede a una ruta. Estas funciones procesan la solicitud y devuelven una respuesta, que suele ser HTML, JSON o una redirección.
  • Plantillas (Templates): Aunque Flask es minimalista, permite usar **Jinja2** (un motor de plantillas) para generar HTML dinámico. Esto significa que puedes tener archivos HTML con "marcadores de posición" que Flask rellena con datos desde el backend antes de enviarlos al navegador.

    render_template(): Función de Flask para renderizar plantillas Jinja2.

  • Solicitud (Request) y Respuesta (Response):
    • Request: Objeto que contiene toda la información de la solicitud HTTP enviada por el cliente (navegador), como datos de formulario, parámetros de URL, encabezados, etc. Puedes acceder a ellos a través del objeto request de Flask.
    • Response: Lo que tu aplicación Flask envía de vuelta al cliente. Puede ser HTML, JSON, imágenes, etc.
  • WSGI (Web Server Gateway Interface): Es una especificación estándar para que los servidores web y las aplicaciones web Python se comuniquen. Flask es una aplicación WSGI. Para desplegar una aplicación Flask en producción, necesitas un servidor WSGI como Gunicorn o uWSGI.
  • Entornos Virtuales (Virtual Environments): ¡Crucial para cualquier proyecto Python! Un entorno virtual aísla las dependencias de tu proyecto de las de otros proyectos y de la instalación global de Python en tu sistema. Esto evita conflictos de versiones y mantiene tus proyectos limpios y manejables.

2. Formas de Instalación mediante CMD (Línea de Comandos) 💻

Antes de instalar Flask, es fundamental que uses un **entorno virtual**.

Paso 1: Instalar Python (Si no lo tienes)

Si aún no tienes Python, descárgalo e instálalo desde el sitio web oficial: python.org. Asegúrate de marcar la opción "Add Python to PATH" durante la instalación.

Paso 2: Crear y Activar un Entorno Virtual

  1. Abre tu Símbolo del Sistema (CMD) o PowerShell.
  2. Navega a la carpeta donde quieres crear tu proyecto. Por ejemplo: 
    cd C:\Users\TuUsuario\Documents\MisProyectosWeb
mkdir mi_aplicacion_flask
cd mi_aplicacion_flask
  3. Crea el entorno virtual: 
    python -m venv venv

    venv es el nombre de la carpeta del entorno virtual.

  4. Activa el entorno virtual:
    • En Windows (CMD): 
      venv\Scripts\activate
    • En Windows (PowerShell): 
      .\venv\Scripts\Activate.ps1

    Verás (venv) al inicio de tu línea de comandos.

Paso 3: Instalar Flask dentro del Entorno Virtual

  1. Con el entorno virtual activado, instala Flask: 
    pip install Flask

Paso 4: Crear tu Primera Aplicación Flask

  1. Crea un archivo llamado app.py en la misma carpeta del proyecto.
  2. Pega el siguiente código en app.py: 
    from flask import Flask

                    app = Flask(__name__)

                    @app.route('/')
                    def hello_world():
                    return '¡Hola, mundo desde mi primera aplicación Flask!'

                    @app.route('/about')
                    def about():
                    return 'Esta es una página de ejemplo sobre nosotros.'

                    if __name__ == '__main__':
                    app.run(debug=True)

Paso 5: Ejecutar la Aplicación Flask

  1. Asegúrate de que tu entorno virtual esté activado.
  2. Ejecuta tu aplicación desde la línea de comandos: 
    python app.py
  3. Verás una salida similar a: 
     * Running on http://127.0.0.1:5000
  4. Abre tu navegador y visita http://127.0.0.1:5000/ o http://127.0.0.1:5000/about.

Desactivar el Entorno Virtual

Cuando termines de trabajar en tu proyecto Flask, puedes desactivar el entorno virtual escribiendo: 

deactivate
